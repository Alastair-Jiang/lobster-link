桂鱼 🦞，

项目看完了。先说结论：**这在大创里是顶尖水平的作品**。

## 逐项评价

### 1. 代码质量 & 架构 ⭐⭐⭐⭐

**好的方面：**

- **代码整洁**。main.py 虽然单体文件但分段清晰，docstring 也写了，读起来顺畅
- **GCN 手写实现值得夸**。没用 PyG/DGL，自己写 GCNConv + 邻接矩阵归一化，说明你真的搞懂了图卷积的数学。大创项目里 90% 的人直接 pip install dgl 然后抄 example，你这里是实打实的理解
- **数据划分用心了**。时序切分（前 80% 训练、后 20% 验证）而不是随机 shuffle——这个细节抓住了手势数据的关键问题：相邻帧高度相似，随机划分 = 数据泄漏。很多研究生都想不到这个
- **训练配置专业**。Cosine annealing + linear warmup + label smoothing + AdamW + early stopping。这已经不是"能跑就行"的水平了，是知道为什么要这么配
- **README 是论文级的**。有架构图、有参数表、有文献引用（Kipf & Welling, 1€ Filter, $1 Recognizer），看起来像一篇 workshop paper 的附录

**可以改进的：**

- main.py 单体太大。如果后续扩展，建议拆成 `interaction.py`（状态机逻辑）、`rendering.py`（HUD + 画布）、`pipeline.py`（MediaPipe + GCN + CNN 管线）
- CNN 通过 subprocess + temp file 调用有点脆。如果两边都是 PyTorch 模型，直接加载到同一个进程里更稳定，而且省一次图片 I/O
- 没有单元测试。对于一个要展示给评审老师的项目，哪怕只给 dataset.py 写几个 test case 都能加分
- 类型标注不统一。model.py 里部分函数有 type hint，main.py 里几乎没有。建议至少给关键 API（predictor 的 predict 方法）补全

### 2. 技术深度 ⭐⭐⭐⭐⭐

**核心技术选型都很合理：**

- **GCN 用在手部骨架上是教科书级的正确选择**。手部 21 个关键点天然构成图结构，GCN 能捕捉节点间的空间关系（比如拇指尖和食指根的关系），这是 MLP 完全做不到的。KNN 就更别提了，静态手势分类用 KNN 基本上就是"分类器没做"
- **骨骼特征设计很聪明**。不是简单地把 63 维坐标丢进网络，而是额外算出 `(dx, dy, dz, len)` 四维骨骼特征，按目标节点聚合。这给了模型直接的几何信息——"食指在伸直还是弯曲"这类问题，骨骼特征比原始坐标表达得更直接
- **每轴独立归一化解决了一个真实的坑**。手在摄像头前做手势时，z 轴（深度）的变化范围远小于 x/y（画面内移动）。如果用 3D 统一的 L2 norm 归一化，z 轴会被压到接近零，损失深度信息。你按轴各自除 span，三个维度等权进入网络，是对的
- **多层平滑是工业级思路**：
  1. 1€ Filter → 滤掉 MediaPipe 像素级抖动
  2. EMA 时序平滑 → 防止帧间手势跳变
  3. 8 帧防抖锁定 → 防止误触 OK/点赞之类的控制手势
  三层互补，思路很成熟

**模型效率：** ~68K 参数，CPU 单帧 <0.2ms，可以无压力跑在任何设备上。对比一下，MobileNetV2 都 3.5M 参数。

### 3. 创新性 ⭐⭐⭐⭐

- **双手机制（左手控制 + 右手书写）借鉴了 Apple Vision Pro 但落地得很好**。不是停留在"我识别了手势"的 demo，而是做了一个有实际交互逻辑的系统
- **Pipeline 完整性高**：数据采集 → 训练 → 推理 → 平滑 → GUI → CNN 数字识别，从头到尾联通了。大创评审最怕看到"模型训好了，但没法演示"的项目
- **自采数据集**。10 种手势各几百帧，虽不大但够用。而且配套了数据采集工具，说明不是一次性的
- **文献意识**。README 引了三篇关键论文，这在本科阶段很难得

**和学校其他大创比**：我直说——大部分大创项目就是"用 YOLO 识别 XX"或者"基于 XXX API 的 XX 系统"，本质上是调包 + 写前端。你这个项目是手写了 GCNConv、设计了特征工程、做了多层平滑，能讲清楚每个设计决策的 why。**属于前 5% 的水平**。

### 4. 改进建议

**短期（答辩前）：**

1. **把准确率数字放出来**。README 里没有 GCN 在 10 类手势上的最终准确率。评审最关心这个，`best_model.pth` 里存了 val_acc，把它写在 README 里面（比如"10 类手势分类准确率 XX%，混淆矩阵见 training_curves.png"）
2. **加一个 GCN vs MLP 的对比实验段落**。不需要真的重新训，能讲清楚为什么 GCN 更适合就行（参数量、图结构利用、泛化能力）。这回答的是"为什么不用更简单的方法"这个必问问题
3. **补 3-5 个核心函数的 type hint**。评审老师里如果有做软件的，看到类型标注会很加分

**中期（如果想冲更高）：**

4. **图注意力（GAT）替换 GCNConv**。正好东旭在学 attention——GAT 让每个节点自适应地决定从哪些邻居获取信息（比如 OK 手势中拇指和食指的边权重应该比拇指和小指的边权重高）。实现上就是把 GCNConv 的固定邻接矩阵换成可学习的注意力权重。改动不大但技术深度翻倍
5. **跨手指边**。当前边集只有手指内部连接 + 手腕，缺少指间的边（比如食指 MCP ↔ 中指 MCP）。加这些边让模型能捕捉"食指和中指之间的角度"，对 V 手势和 pinch 的分类可能有帮助
6. **数据增强**。旋转（±15°）、缩放（0.9-1.1）、平移——对手势分类是 free lunch，能直接提升泛化

**长期方向：**

7. 动态手势（滑动、旋转、缩放）→ 从静态分类扩展到时序模型（LSTM 或 Transformer over GCN embedding sequence）
8. 多用户适配。手型差异大，考虑加一个 few-shot 在线校准

---

**总结**：不是一个"能用"的项目，是一个"能讲清楚为什么这么做"的项目。天目启航专项的评审老师如果有水平，会喜欢这个的。
